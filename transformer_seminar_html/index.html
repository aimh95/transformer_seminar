Transformer는 문장을 어떻게 번역하는가?

LLM을 처음 보는 팀원을 위한 Encoder-Decoder 구조 설명

예제: "The cat is on the mat." → "Le chat est sur le tapis."

LLM은 입력을 받아 어떻게 출력을 만들까?

입력 문장이 token으로 바뀌고, 문맥을 반영한 뒤, Decoder가 다음 token을 하나씩 예측해 출력 문장을 만듭니다.

1

Transformer는 왜 나왔는가?

Transformer(Vaswani et al., 2017)는 기존 RNN 기반 번역 모델의 한계를 극복하기 위해 제안되었습니다.

기존 RNN의 한계

  • 문장을 왼쪽부터 한 단어씩 순서대로 처리 → 병렬화 불가
  • 멀리 떨어진 단어 간 관계가 흐려짐 (장거리 의존성 문제)
  • 문장이 길수록 앞쪽 정보가 점점 희석됨

Transformer의 해결책

  • Attention으로 모든 token 간 관계를 직접 계산
  • 전체 문장을 한 번에 처리 → GPU 병렬 연산 가능
  • 멀리 떨어진 단어도 Attention 한 번으로 직접 연결
비유: RNN은 문장을 왼쪽부터 한 단어씩 읽는 방식에 가깝습니다. Transformer는 문장 전체를 펼쳐놓고 단어들 사이의 관계를 한 번에 계산하는 방식에 가깝습니다.
한 줄 요약 Transformer = Attention 기반 Encoder-Decoder. 순차 처리 없이 전체 문장을 병렬 처리하고, 멀리 떨어진 token도 직접 참조할 수 있다.
2

Tokenizer — 문장을 숫자 ID로

컴퓨터는 문자를 직접 처리할 수 없습니다. Tokenizer는 문장을 token으로 나누고, 각 token을 vocabulary에서 정수 ID로 변환합니다.

예시 — 단어 단위 Tokenization
"The cat is on the mat."
→ ["The", "cat", "is", "on", "the", "mat"]
→ [1, 5, 2, 6, 1, 10 ]
token id는 모델이 의미를 이해하는 값이 아니라 vocabulary에서 특정 token을 가리키는 index입니다.
실제 모델: 단어 단위가 아니라 BPE(Byte-Pair Encoding) 같은 subword 단위로 분할하는 경우가 많습니다. 예를 들어 "playing" → ["play", "##ing"]. 이 세미나에서는 이해를 위해 단어 단위로 단순화합니다.
Padding: 배치(batch) 단위로 여러 문장을 한 번에 처리할 때, 길이가 짧은 문장 뒤에 <PAD>(id=0)를 붙여 길이를 맞춥니다. Padding Mask가 이 위치를 Attention에서 무시하도록 처리합니다.
한 줄 요약 Tokenizer = 텍스트 → 정수 ID 배열. token id는 단어의 의미가 아니라 vocabulary 인덱스일 뿐이다.
3

Embedding — token ID를 벡터로

token id는 단순한 숫자 label이므로 모델이 의미를 이해할 수 없습니다. Embedding Layer는 각 token id를 d_model 차원의 실수 벡터로 변환합니다.

Embedding Lookup
W_emb ∈ ℝ^{V × d_model}
E_i = W_emb[token_id_i]
V = vocabulary 크기, d_model = 512 (논문 기본값). Embedding은 W_emb의 특정 row를 가져오는 lookup 연산입니다.
예시: token id가 [1, 5, 2, 6, 1, 10]이라면 → W_emb의 1번, 5번, 2번, 6번, 1번, 10번 row를 순서대로 가져옵니다. 결과 shape: (6, 512)

Embedding vector는 학습 과정에서 의미적으로 유사한 단어가 비슷한 방향을 갖도록 업데이트됩니다. "cat"과 "dog"의 벡터는 가깝고, "cat"과 "Paris"의 벡터는 다른 방향에 위치하게 됩니다.

핵심: 같은 token id는 항상 같은 embedding vector를 가집니다. "The"(위치 0)와 "the"(위치 4)가 같은 id라면 같은 embedding입니다. 위치 차이는 다음 단계인 Positional Encoding에서 처리합니다.
한 줄 요약 Embedding = token id → 512차원 실수 벡터. 학습 가능한 lookup table. 같은 id = 같은 벡터.
4

Positional Encoding — 순서 정보 주입

Transformer는 RNN처럼 순서대로 처리하지 않고 전체 sequence를 한 번에 처리합니다. 따라서 모델 스스로는 token의 위치 순서를 알 수 없습니다. Positional Encoding(PE)을 embedding에 더해 위치 정보를 전달합니다.

Encoder 입력
Input = Embedding + PositionalEncoding
Embedding과 PE는 모두 shape (seq_len, d_model). 같은 shape이라 원소별 덧셈(element-wise add) 가능.
예시: "The"(위치 0)와 "the"(위치 4)는 같은 token id → 같은 Embedding 벡터 E₁.
하지만 PE(pos=0) ≠ PE(pos=4) 이므로
→ E₁ + PE(pos=0) ≠ E₁ + PE(pos=4)
→ 모델에는 다른 입력으로 들어간다.
Advanced: sin/cos Positional Encoding 수식
논문 수식 (arXiv:1706.03762)
PE(pos, 2i) = sin(pos / 10000^{2i/d_model})
PE(pos, 2i+1) = cos(pos / 10000^{2i/d_model})
pos = token 위치 (0, 1, 2...), i = 차원 인덱스 (0 ~ d_model/2-1)
학습 파라미터 없이 수식으로 결정. 긴 문장에도 외삽(extrapolation) 가능.
Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.
한 줄 요약 PE = 위치마다 다른 sin/cos 패턴. Embedding에 더해 순서 정보를 모델에 전달한다. 학습 파라미터 없음.
5

Encoder-Decoder 전체 구조

논문 기준으로 Encoder는 6개의 Encoder Layer로 구성되고, Decoder도 6개의 Decoder Layer로 구성됩니다.

Encoder

입력: "The cat is on the mat." (source)

역할: source 문장을 읽고 내부 표현 H_enc 생성

H_enc shape: (src_len, d_model) = (6, 512)

Decoder

입력: H_enc + 이전 target token

역할: H_enc를 참고하며 번역 문장 생성

출력: 다음 token의 확률 분포

핵심 연결: Encoder가 만든 H_enc는 Decoder의 Cross-Attention에서 Key, Value로 사용됩니다. Encoder와 Decoder를 연결하는 유일한 경로입니다.
한 줄 요약 Encoder(×6) → H_enc → Decoder(×6). Encoder는 source를 압축하고, Decoder는 그것을 참고해 target을 생성한다.
6

Encoder Layer 구조

각 Encoder Layer는 두 개의 sub-layer로 구성됩니다. 각 sub-layer 뒤에는 Residual Connection + Layer Normalization이 적용됩니다.

  1. Multi-Head Self-Attention: source 문장의 각 token이 다른 모든 token을 참고한다. "cat"은 "mat", "is", "The" 등과 관계를 계산한다.
  2. Add & Norm: Residual Connection + Layer Normalization. LayerNorm(x + SubLayer(x))
  3. Feed Forward Network: 각 token을 독립적으로 비선형 변환한다.
  4. Add & Norm: 다시 Residual + LayerNorm.
Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.
이 구조가 6번 반복됩니다. 각 레이어는 독립적인 파라미터(W^Q, W^K, W^V, W₁, W₂)를 가집니다. 입력부터 출력까지 shape이 (seq_len, d_model)로 유지됩니다.
한 줄 요약 Encoder Layer = Multi-Head Self-Attention → Add&Norm → FFN → Add&Norm. 이 구조가 ×6 쌓여 H_enc를 만든다.
7

Q, K, V 생성 — 같은 X에서 세 가지 투영

Self-Attention의 시작은 입력 X에서 Query(Q), Key(K), Value(V)를 만드는 것입니다. 각각 다른 학습 가능한 weight matrix를 곱해 선형 투영합니다.

Q
Query — 무엇을 찾는가
현재 위치가 어떤 정보를 원하는지 나타내는 벡터
비유: 검색어 (검색하는 쪽)
K
Key — 어떤 특징을 갖는가
각 token이 검색될 때 드러내는 특징 벡터
비유: 문서 태그 (검색 대상)
V
Value — 실제 전달할 정보
Q-K 매칭 후 실제로 가져오는 내용 벡터
비유: 실제 문서 내용
Q, K, V 수식
Q = X W^Q, shape: (seq_len, d_k)
K = X W^K, shape: (seq_len, d_k)
V = X W^V, shape: (seq_len, d_v)
X ∈ ℝ^{seq_len × d_model}, W^Q, W^K ∈ ℝ^{d_model × d_k}, W^V ∈ ℝ^{d_model × d_v}
논문 기본값: d_model=512, d_k=d_v=64 (head당)
Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.
중요: Q/K/V의 역할("검색어", "태그", "내용")은 처음부터 설계자가 정한 것이 아닙니다. 학습 초기의 W^Q, W^K, W^V는 랜덤 초기화이므로 Q, K, V에 아무 의미가 없습니다. loss를 줄이는 학습 과정에서 Attention 수식 안의 각 위치에 맞는 역할로 수렴하게 됩니다.
한 줄 요약 Q, K, V = 같은 X를 다른 W로 선형 변환. 역할은 학습으로 결정된다. Q는 무엇을 찾는가, K는 어떤 특징인가, V는 실제로 전달할 내용.
8

Attention Score Matrix — 토큰 간 관련도

Q와 K의 내적(dot product)으로 Attention Score Matrix를 계산합니다. 이 행렬은 "i번째 token이 j번째 token을 얼마나 참고해야 하는가"를 나타냅니다.

Attention Score Matrix
Score = Q · Kᵀ shape: (seq_len, seq_len)
score_ij = q_i · k_j
Q shape: (seq_len, d_k), Kᵀ shape: (d_k, seq_len) → Score: (seq_len, seq_len)
row i = "i번째 Query가 모든 Key에 대해 계산한 점수"
Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.
비유: Query는 검색어, Key는 문서의 태그, Value는 실제 문서 내용에 가깝습니다. 검색어와 태그가 잘 맞을수록 그 문서 내용을 더 많이 가져옵니다.
score_ij가 크다 = "i번째 token이 j번째 token에 주목한다"
한 줄 요약 Score = QKᵀ, shape (seq_len, seq_len). score_ij = q_i · k_j = i번째가 j번째를 얼마나 참고할지의 원시 점수.
9

Scaled Dot-Product Attention

공식 (논문 Eq. 1)
Attention(Q, K, V) = softmax( QKᵀ / √d_k ) · V
  1. Q와 K 내적: Score = QKᵀ → 각 token 쌍의 관련도 점수
  2. √d_k로 나누기: Score 값의 분산을 안정화
  3. Softmax: 각 row를 확률 분포로 변환 (합 = 1.0)
  4. V와 곱하기: 확률 비율에 따라 Value 정보를 가중합
Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.
√d_k로 나누는 이유: Q와 K의 내적은 d_k개 항의 합입니다. d_k가 크면 내적값의 분산이 d_k에 비례해 커집니다. 값이 너무 크면 Softmax가 한 위치에만 확률을 몰아주고 gradient가 사라집니다. √d_k로 나누면 분산이 안정화됩니다.

주의: sequence 길이가 길어서 나누는 것이 아닙니다. 원인은 d_k 차원 크기입니다.
한 줄 요약 Attention = softmax(QKᵀ/√d_k)·V. 점수 계산 → 스케일링 → 확률화 → Value 가중합. 이 4단계가 전부다.
10

Multi-Head Attention — 여러 관점으로 보기

하나의 Attention만으로는 한 가지 관점의 관계만 포착합니다. Multi-Head Attention은 d_model 차원을 h개의 head로 나누어 각 head가 서로 다른 관계 패턴을 학습하게 합니다.

논문 기본값
d_model = 512, h = 8 heads
d_k = d_v = d_model / h = 512 / 8 = 64

head_i = Attention(Q_i, K_i, V_i)
MultiHead(Q,K,V) = Concat(head_1,...,head_8) W^O
W^O ∈ ℝ^{512×512}. Concat 후 W^O로 다시 d_model=512로 변환.
Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.
자주 하는 오해: Multi-Head Attention이 sequence를 token 단위로 나누는 것이 아닙니다. 각 token representation의 d_model 차원을 head 수로 나누어 여러 Attention을 병렬 수행하는 구조입니다. 모든 head는 전체 sequence를 처리합니다.
비유: 같은 문장을 여러 명의 관찰자가 서로 다른 관점으로 보는 것과 비슷합니다. 어떤 head는 문법적 관계를, 어떤 head는 의미적 관계를, 어떤 head는 먼 token과의 관계를 포착할 수 있습니다. (단, 역할이 반드시 이렇게 나뉜다는 것이 아니라, 그런 식으로 역할이 분화될 수 있습니다.)
한 줄 요약 Multi-Head = d_model 차원을 h등분 → 각 head가 독립 Attention → Concat → W^O. 다양한 관계 패턴을 동시에 학습한다.
11

Residual Connection과 Layer Normalization

각 sub-layer(MHA, FFN) 뒤에는 반드시 Residual Connection + Layer Normalization이 적용됩니다.

Add & Norm 공식
Output = LayerNorm(x + SubLayer(x))
x = sub-layer 입력, SubLayer(x) = MHA 또는 FFN 출력. x와 SubLayer(x)의 shape이 반드시 같아야 합니다.

Residual Connection

원래 입력 x를 sub-layer 출력에 더해주는 '지름길'입니다.

  • 깊은 네트워크에서 gradient 소실 방지
  • x와 SubLayer(x) shape이 같아야 더할 수 있음 → Transformer 전체에서 d_model=512가 유지되는 이유

Layer Normalization

각 token의 d_model 차원에 걸쳐 정규화합니다. 학습 안정화 효과.

  • 평균=0, 분산=1로 정규화
  • batch 크기와 무관하게 동작 (Batch Norm과 다른 점)
한 줄 요약 Add = 원본 x를 더해 gradient 흐름 보장. Norm = 학습 안정화. x + SubLayer(x)의 shape이 같아야 하므로 d_model이 일정하게 유지된다.
12

Feed Forward Network — 토큰별 독립 변환

Attention이 "어떤 token들을 참고할지" 결정했다면, FFN은 "각 token의 표현을 어떻게 변환할지" 결정합니다. token 간 정보 교환 없이 각 위치를 독립적으로 처리합니다.

FFN 공식
FFN(x) = max(0, x W₁ + b₁) W₂ + b₂
W₁ ∈ ℝ^{512×2048}, W₂ ∈ ℝ^{2048×512}
512 → 2048 (ReLU) → 512 (d_ff = 2048 = 4×d_model)
Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.
Attention vs FFN 역할 비교
Attention: token 간 정보를 섞는다 — "어디서 정보를 가져오는가?"
FFN: 섞인 정보를 각 token 위치에서 독립적으로 가공한다 — "가져온 정보를 어떻게 변환하는가?"
마지막에 d_model로 돌아오는 이유: Residual Connection 때문입니다. LayerNorm(x + FFN(x))에서 x와 FFN(x)를 더하려면 shape이 같아야 합니다. 내부에서는 2048로 확장하더라도 최종 출력은 반드시 512로 축소합니다.
한 줄 요약 FFN = 각 token을 독립적으로 512→2048→512 MLP 변환. token 간 정보를 새로 섞지 않는다.
13

Decoder 구조 — 번역 문장 생성

Decoder Layer는 세 개의 sub-layer로 구성됩니다. Encoder Layer보다 Cross-Attention이 하나 더 있습니다.

  1. Masked Multi-Head Self-Attention: target 문장 내부 token들이 서로 참고하되, 미래 token은 볼 수 없도록 Mask를 적용한다.
  2. Add & Norm
  3. Cross-Attention: Decoder의 현재 상태(Q)가 Encoder 출력 H_enc(K, V)를 참고한다. source 문장의 어디를 볼지 결정하는 단계.
  4. Add & Norm
  5. Feed Forward Network: 각 token을 독립 변환.
  6. Add & Norm
한 줄 요약 Decoder = Masked Self-Attn + Cross-Attn + FFN. Encoder와 달리 Cross-Attention이 추가되어 source 문장을 참고한다.
14

학습 시 Decoder 입력: Shifted Target

학습 시에는 정답 target 문장을 알고 있습니다. 정답 문장을 오른쪽으로 한 칸 shift하고 앞에 <SOS>를 붙여 Decoder 입력으로 사용합니다. 이를 Teacher Forcing이라 합니다.

Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.
Teacher Forcing 예시
Target: [Le, chat, est, sur, le, tapis]
입력: [<SOS>, Le, chat, est, sur, le ]
예측 목표: [Le, chat, est, sur, le, tapis]
Decoder는 현재까지의 target token을 보고 다음 token을 맞히도록 학습됩니다.
핵심: 학습 시에는 전체 shifted target sequence가 한 번에 입력되어 모든 위치를 병렬 계산합니다. 그러나 Look-Ahead Mask 때문에 각 위치는 미래 token을 볼 수 없습니다. → 학습 속도가 RNN보다 빠른 이유.
한 줄 요약 학습 시 Decoder 입력 = 정답 문장을 <SOS> 추가 후 right-shift. 전체를 병렬 계산하되, Mask로 미래는 차단한다.
15

Look-Ahead Mask — 미래 차단

Decoder는 번역 문장을 생성할 때 아직 생성하지 않은 미래 token을 미리 볼 수 없습니다. Look-Ahead Mask는 이를 강제합니다.

중요: Mask는 Q, K, V 자체에 적용하는 것이 아닙니다. QKᵀ로 만든 Attention Score Matrix에 더합니다.
  1. Q = X_dec W^Q
  2. K = X_dec W^K
  3. V = X_dec W^V
  4. Score = Q · Kᵀ
  5. MaskedScore = Score + Mask ← 이 단계에서 Mask 적용
  6. AttentionWeight = softmax(MaskedScore / √d_k)
  7. Output = AttentionWeight × V
Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.

Cross-Attention — Encoder와 Decoder의 연결

Decoder의 두 번째 sub-layer는 Cross-Attention입니다. Q는 Decoder에서, K와 V는 Encoder 출력 H_enc에서 옵니다.

Cross-Attention 수식
Q_cross = H_dec W^Q ← Decoder hidden state
K_cross = H_enc W^K ← Encoder output
V_cross = H_enc W^V ← Encoder output
Score shape: (tgt_len, src_len) = (4, 6). "번역 4개 token이 원본 6개 token을 얼마나 참고하는가"
Redrawn based on "Attention Is All You Need", Vaswani et al., 2017.
예시: "chat"을 예측해야 할 때, source 문장의 "cat" 정보가 중요합니다. 학습이 잘 된 모델에서는 Decoder Query(chat 위치)가 Encoder의 "cat" Key와 높은 score를 가지도록 학습됩니다.

→ "cat" 위치의 Attention Weight가 커짐 → "cat" Value 정보가 Decoder에 더 많이 반영됨 → "chat" 예측에 도움
한 줄 요약 Cross-Attention = Decoder(Q)가 Encoder(K, V)를 참고. "지금 생성할 번역 단어를 위해 source 문장 어디를 참고할지" 결정한다.
17

Padding Mask — <PAD> 위치 무시

배치(batch) 단위로 여러 문장을 처리할 때 길이를 맞추기 위해 짧은 문장 뒤에 <PAD>를 붙입니다. Padding Mask는 <PAD> 위치에 Attention이 분배되지 않도록 합니다.

Padding Mask 예시
배치 내 문장 1: [The, cat, is, on, the, mat, <PAD>, <PAD>]
배치 내 문장 2: [A, dog, runs, fast, here, in, the, park]

Mask: <PAD> 위치의 Score를 −∞로 설정
<PAD> 토큰은 실제 의미가 없으므로 Attention에서 무시합니다.

Padding Mask 적용 위치

  • Encoder Self-Attention
  • Decoder Masked Self-Attention
  • Decoder Cross-Attention

Look-Ahead Mask와의 차이

  • Padding Mask: 실제 token이 아닌 <PAD>를 막는다
  • Decoder에서는 두 Mask를 결합해서 사용한다
한 줄 요약 Padding Mask = <PAD> 위치 Score를 −∞로. Look-Ahead Mask = 미래 위치 차단. 둘 다 Score Matrix에 더해 작동한다.
18

Linear + Softmax — 다음 token 확률 계산

Decoder의 최종 출력은 Linear Layer를 거쳐 vocabulary 크기의 logit vector로 변환되고, Softmax로 각 token의 확률을 계산합니다.

Linear + Softmax
Logits = H_dec · W_vocab + b shape: (tgt_len, V)
P(y_t | y_<t, x) = softmax(Logits)
W_vocab ∈ ℝ^{d_model × V}. V = vocabulary 크기. 가장 높은 확률의 token을 선택하거나 beam search 등을 사용.

예를 들어 "chat"을 예측해야 하는 위치에서:

token확률 (예시)bar
Le0.01
chat0.82
est0.05
tapis0.02

학습 시

정답 token의 확률을 높이는 방향으로 loss를 계산합니다.

Loss = −log P(y_t)

추론 시

확률이 가장 높은 token을 선택(greedy)하거나, beam search로 더 좋은 번역을 탐색합니다.

한 줄 요약 Linear = d_model → vocabulary 크기 logit. Softmax = 각 token 확률. 가장 높은 확률 token을 선택하거나 beam search 사용.

전체 요약

  1. Tokenizer는 문장을 token id로 바꾼다.
  2. Embedding은 token id를 d_model 차원 벡터로 바꾼다.
  3. Positional Encoding은 위치 정보를 더한다.
  4. Encoder는 source 문장을 문맥이 반영된 표현(H_enc)으로 바꾼다.
  5. Attention은 token들이 서로를 얼마나 참고할지 계산한다.
  6. FFN은 각 token 표현을 독립적으로 가공한다.
  7. Decoder는 이전 target token과 H_enc를 참고해 다음 token을 예측한다.
  8. Mask는 미래 token이나 <PAD> token을 참고하지 못하게 막는다.
  9. Linear + Softmax는 다음 token 확률 분포를 만든다.
Transformer를 한 문장으로 정리하면:
Transformer는 token 간 관계를 Attention으로 계산하고, FFN으로 각 token 표현을 가공하면서, 입력 문장을 출력 문장으로 변환하는 Encoder-Decoder 구조입니다.
