강의 자료 · Attention 수식의 역사적 배경

Attention 수식은 어떻게 만들어졌나

Seq2Seq의 고정 벡터 병목에서 Softmax(QKᵀ/√d_k)V까지, 2014~2017년의 흐름

Transformer의 attention 수식은 어느 날 갑자기 등장한 것이 아니다. Seq2Seq의 병목 → Bahdanau의 soft alignment → Luong의 dot-product score → Memory Networks의 Key/Value 분리 관점이 차례로 쌓여 지금의 형태로 수렴했다. 이 자료는 그 흐름을 시대순으로 따라가며, 수식의 각 부분이 왜 지금 모습이 되었는지를 설명한다.

1

Attention 이전: Seq2Seq의 고정 벡터 병목

Attention 이전의 대표적인 NMT 구조는 RNN 기반 Encoder-Decoder였습니다. Encoder RNN이 입력 sequence를 읽고 마지막 hidden state, 즉 고정 길이 vector 하나로 문장 전체를 압축한 뒤, Decoder RNN이 그 vector만 보고 target sequence를 생성하는 구조였습니다.

문제: 문장 전체의 모든 정보가 vector 하나에 압축되어야 합니다. 짧은 문장은 괜찮지만, 긴 문장에서는 앞부분·중간·뒷부분의 정보가 모두 같은 vector에 들어가야 합니다. Decoder가 target word를 생성할 때마다 필요한 source 정보는 다른데, 매번 같은 vector 하나만 보고 생성해야 하는 병목이 생깁니다.
이 병목에서 나온 질문: "Decoder가 target word를 만들 때마다, source sentence의 필요한 위치를 직접 다시 보게 할 수 없을까?" 이 질문이 Attention의 출발점입니다.
참고 논문: Cho, K. et al. (2014). Learning Phrase Representations using RNN Encoder-Decoder for Statistical Machine Translation. arXiv:1406.1078.  ·  Sutskever, I., Vinyals, O., & Le, Q. V. (2014). Sequence to Sequence Learning with Neural Networks. NeurIPS.
2

Bahdanau Attention — Soft Alignment의 시작 (2014)

Bahdanau, Cho, Bengio의 논문 제목은 Neural Machine Translation by Jointly Learning to Align and Translate입니다. 번역(translate)만 하는 게 아니라 align과 translate를 함께 학습하자는 제안이었습니다. 고정 vector 병목을 해결하기 위해, target word를 예측할 때마다 source sentence의 관련 부분을 자동으로 soft-search하도록 만들었습니다.

아직 Q, K, V라는 이름은 없지만, 구조는 이미 "score 계산 → softmax로 비율화 → weighted sum"이었습니다.

Step 1 — Alignment score (작은 feed-forward network)
\[ \begin{aligned} e_{ij} &= a(s_{i-1},\, h_j) \\ &= v_a^\top \tanh\!\big(W_a s_{i-1} + U_a h_j\big) \end{aligned} \]
$a(\cdot)$는 추상화된 표기일 뿐, 안에는 작은 MLP가 들어 있다 — $s_{i-1}$과 $h_j$를 각각 선형변환($W_a$, $U_a$)해서 더하고, $\tanh$를 거친 뒤, $v_a^\top$로 다시 한 번 선형 projection해서 점수 하나를 만든다.
$s_{i-1}$: Decoder의 이전 hidden state — "지금 무엇이 필요한가"  ·  $h_j$: Encoder의 $j$번째 hidden state — "참고 후보 + 실제 내용"
Step 2 — Softmax로 비율화
\[ \alpha_{ij} = \dfrac{\exp(e_{ij})}{\sum_{k} \exp(e_{ik})} \]
Step 3 — Context vector = weighted sum
\[ c_i = \sum_{j} \alpha_{ij}\, h_j \]

역할의 싹은 있었지만, 분리되지는 않았다

$s_{i-1}$은 이후 Query가 맡을 역할(지금 필요한 정보)에 가깝고, $h_j$는 이후 Key와 Value가 나눠 맡을 역할을 동시에 했습니다. 관련도 계산에도 쓰이고, context vector를 만들 때도 그대로 쓰였습니다.

왜 argmax가 아니라 softmax였나

통계적 번역에서는 "이 target word는 source의 몇 번째 word와 대응"하는 hard alignment가 흔했습니다. 그러나 argmax로 하나를 고르면 미분 가능하지 않아 gradient descent로 학습할 수 없습니다. Softmax는 모든 source word를 비율적으로 참고하는 미분 가능한 soft alignment를 만들기 위한 선택이었습니다.

핵심: Softmax는 "확률처럼 보기 좋아서" 들어간 게 아니라, hard alignment를 neural network 안에서 학습 가능한 soft alignment로 바꾸기 위해 들어간 장치입니다. 이 정신은 Transformer의 $\mathrm{Softmax}(\textcolor{#C53030}{Q}\textcolor{#2B6CB0}{K}^\top)$까지 그대로 이어집니다.
참고 논문: Bahdanau, D., Cho, K., & Bengio, Y. (2014/2015). Neural Machine Translation by Jointly Learning to Align and Translate. arXiv:1409.0473.
3

Luong Attention — Dot-Product Score의 등장 (2015)

Luong, Pham, Manning은 global attention과 local attention, 그리고 여러 alignment(score) 함수를 비교했습니다. 여기서 중요한 변화는 score를 꼭 작은 신경망(MLP)으로 계산하지 않아도 된다는 것이었습니다.

Bahdanau식 (additive / concat) — 작은 MLP로 계산
\[ \text{score}(h_t, h_s) = v_a^\top \tanh\!\big(W_a [h_t; h_s]\big) \]
Luong이 추가한 두 가지 — 행렬곱만으로 계산
\[ \text{score}(h_t, h_s) = h_t^\top h_s \quad \text{(dot)} \]
\[ \text{score}(h_t, h_s) = h_t^\top W_a\, h_s \quad \text{(general)} \]
$h_t$: decoder의 현재 hidden state  ·  $h_s$: encoder의 source hidden state
역사적으로 중요한 전환: "Attention score를 꼭 작은 MLP로 계산하지 않아도 된다. 두 vector의 내적만으로도 compatibility를 계산할 수 있다." Dot product는 행렬곱으로 매우 빠르게 병렬 계산할 수 있어, 나중에 RNN을 없애고 모든 token 쌍의 attention을 한 번에 계산하려는 시도에 훨씬 유리합니다.

즉 Luong attention은 "Softmax(score) × hidden states"에서 score 계산 방식을 다음 방향으로 열어준 단계입니다.

\[ \text{score} = \mathrm{MLP}(s, h) \]
Bahdanau (2014)
↓
\[ \text{score} = \text{vector}_1^\top \cdot \text{vector}_2 \]
Luong (2015) — 이후 $QK^\top$로 일반화
참고 논문: Luong, M., Pham, H., & Manning, C. D. (2015). Effective Approaches to Attention-based Neural Machine Translation. arXiv:1508.04025.

Memory Networks와 Key-Value 분리 관점 (2014–2016)

Attention은 번역 쪽 alignment 문제와는 별개로, memory를 읽는 방식으로도 발전했습니다. Weston et al.의 Memory Networks와 Graves et al.의 Neural Turing Machines는 long-term memory를 두고, attentional process로 memory를 읽고 쓰는 모델을 end-to-end differentiable하게 학습할 수 있다고 제안했습니다.

여기서 attention은 "현재 상태로 memory의 어느 slot을 읽을지 정하고, 그 slot의 내용을 가져온다"로 해석됩니다 — Transformer의 QKV 직관과 매우 가깝습니다.

이전 attention: 역할이 한 벡터에 겸용

$h_j$ = 검색될 때 쓰이는 표현 = 실제로 가져올 내용

Bahdanau/Luong 모두 encoder hidden state 하나가 두 역할을 동시에 수행했습니다.

Key-Value Memory Networks (2016): 역할 분리

Key 검색될 때 쓰이는 주소/색인  ·  Value 실제로 가져올 내용

Document QA를 위해 addressing 단계와 output 단계에 서로 다른 encoding을 쓰는 구조를 제안했습니다.

과도한 단순화 주의: Transformer가 Key-Value Memory Networks를 그대로 가져온 것은 아닙니다. 다만 "attention을 memory read로 보는 관점", "addressing과 output을 분리하는 관점"이 이미 존재했고, Transformer는 이 관점을 sequence modeling 안의 깔끔한 행렬 연산으로 정리한 것이라고 보는 것이 정확합니다.
참고 논문: Weston, J., Chopra, S., & Bordes, A. (2014). Memory Networks. arXiv:1410.3916.  ·  Graves, A., Wayne, G., & Danihelka, I. (2014). Neural Turing Machines. arXiv:1410.5401.  ·  Miller, A. et al. (2016). Key-Value Memory Networks for Directly Reading Documents. arXiv:1606.03126.

Transformer의 종합 — 왜 QKᵀ, Softmax, V, √d_k인가

Transformer의 선언은 제목 그대로 Attention Is All You Need입니다. RNN이나 CNN 없이 attention만으로 sequence transduction을 하겠다는 것이었고, 이를 위해 attention을 decoder time step 루프가 아니라 하나의 큰 행렬 연산으로 다시 써야 했습니다.

한 token씩 계산하면 (RNN attention과 동일한 형태)
\[ q_i \cdot k_1,\ \ q_i \cdot k_2,\ \ \dots,\ \ q_i \cdot k_n \]
모든 token에 대해 한 번에 계산하면 (Transformer)
\[ \textcolor{#C53030}{Q}\textcolor{#2B6CB0}{K}^\top \]

그래서 QKᵀ는 "수학자가 예뻐서 만든 행렬곱"이 아니라, 모든 token이 모든 token을 병렬로 검색해야 해서 생긴 계산 형태입니다.

앞선 세 갈래의 흐름이 최종 수식 한 줄로 수렴합니다.

시대 (연도)핵심 기여Transformer 수식에서의 위치

↓
\[ \mathrm{Softmax}\!\left(\dfrac{\textcolor{#C53030}{Q}\textcolor{#2B6CB0}{K}^\top}{\sqrt{d_k}}\right) \cdot \textcolor{#276749}{V} \]
Transformer, 2017 — 분산 안정화 추가

Softmax가 남은 이유

Bahdanau 때부터 attention의 본질은 hard selection이 아니라 soft selection이었습니다. Transformer에서도 이 정신은 그대로 유지됩니다 — 여러 token을 비율적으로 참고하는 미분 가능한 구조입니다.

V를 곱하는 이유

Bahdanau의 context vector $c_i = \sum_j \alpha_{ij} h_j$는 Transformer에서 $\sum_j \alpha_{ij} v_j$가 됩니다. 검색용 표현(Key)과 전달용 표현(Value)을 분리했기 때문에, hidden state 대신 Value를 가중합합니다.

√d_k가 마지막에 붙은 이유: dot-product attention은 additive attention과 이론적 복잡도는 비슷하지만, 행렬곱으로 훨씬 빠르고 memory-efficient합니다. 다만 $d_k$가 커지면 내적의 분산이 커져 softmax가 매우 작은 gradient 영역으로 밀릴 수 있습니다. 그래서 Transformer는 dot-product attention을 큰 차원에서도 안정적으로 쓰기 위한 보정항으로 $\sqrt{d_k}$를 마지막에 추가했습니다.
참고 논문: Vaswani, A. et al. (2017). Attention Is All You Need. arXiv:1706.03762.
